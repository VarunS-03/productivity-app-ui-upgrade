Hunter System — UX Audit

Solo Leveling RPG Quest Tracker · Full product teardown

Context analysis

What this app is
A Solo Leveling manhwa-themed gamified productivity tracker. Users log daily tasks as "quests", earn XP and Gold, level up, maintain login streaks, unlock dungeons, and purchase items with earned gold. It is entirely local/offline-first — save data is JSON imported/exported. There is no backend or account system.
Target user
Solo Leveling fans aged 16–28. Productivity seekers who want gamification. Heavy mobile users. Likely have tried and abandoned habit apps before.
Immediate user goal
Log today's tasks, hit Daily Login, complete quests, see XP/level progress, not break their streak.
What business success looks like
Daily active usage (DAU), streak retention, return rate after day 3 (the critical habit cliff), and organic sharing through the RPG identity the user builds. If monetized later: converting engaged users to premium via gold economy or cosmetics.

Visual system flaws

Critical
Header has 9 competing elements — zero hierarchy
The HUD header contains: brand ("NEXUS⬡SYS"), LVL stat, STRK stat, a spacer, hunter name, system status, a second streak display, EOD countdown, save indicator, and bonus gate button. That is 9–10 distinct UI objects on one line. No visual grouping, no size hierarchy. On a 375px phone this is illegible noise. Streak is duplicated: hud-strk and header-streak both exist.
Critical
"SYSTEM OFFLINE" as the default state label is alarming
Before daily login, header-status displays "SYSTEM OFFLINE". To any new user, this looks like a broken app. In a gamified context, negative system-state language creates anxiety, not engagement. Apps like Duolingo never tell you the system is broken — they always present an inviting, ready state.
High
4 typefaces — far too many for coherent design
Black Ops One, Rajdhani, Bebas Neue, and Share Tech Mono are all loaded. No top-tier product uses more than 2 typefaces. Each additional font fractures visual identity and increases load time. Pick two: one display font for headers/brand moments, one workhorse for UI copy.
High
ALL CAPS throughout — fatiguing for body text
ALL CAPS works for badge labels and nav items. Used for multi-sentence descriptions ("Forged in the abyss. Increases quest XP by 10% when equipped. Binds to Shadow Monarch lineage."), it becomes physically uncomfortable to read. Mixed-case prose inside an ALL CAPS UI shell would give the app more visual range and readability.
High
Navigation labels are cryptic abbreviations
ARS, HQ, QST, DNG — none of these are universally understood. "HQ" reads as home base in military/corporate contexts, but "ARS" as Arsenal is not intuitive. A new user seeing these 4 tabs has no mental model of the app structure. Add icons and use full words or clear short labels: Shop, Home, Quests, Dungeon.
Medium
Navigation is placed mid-screen, not at the bottom
On mobile, the thumb's natural reach is the bottom of the screen. The nav sitting between the header and content forces awkward two-handed use. iOS and Android both have established bottom-tab-bar patterns for a reason. This app is viewport-fit:cover, confirming it's mobile-first — yet ignores the most fundamental mobile navigation pattern.
Medium
Inline styles scattered throughout — broken design system
Dozens of style="margin-top:16px", style="width:100%", style="font-size:8px" inline overrides throughout the HTML. Multiple button archetypes: sys-btn, buy-btn, cmd-btn, gate-open-btn, enter-btn. No single source of truth for spacing or component state.
Low
Empty drag-drop zones make the app look unfinished
Every tab has large "DRAG IMAGE HERE" placeholder zones for manhwa art. Until images are loaded, the app looks like a wireframe. This is an intentional customization feature, but the empty-state presentation needs a more atmospheric fallback — a low-opacity procedural gradient or SVG illustration — so the app feels polished by default.

UX + flow analysis

Critical
The HQ tab is doing the work of 4 screens
Sections on one tab: Awakening Gate + Onboarding Panel + Mission Control + Quick Register + Hero Image Drop + Status Window (with 5 sub-bars) + Commands + Activity Log + EOD Summary + Achievements + Skill Tree + Data Management. A user lands here and has no idea where to focus. This isn't "headquarters" — it's a storage closet. The critical action (Daily Login) is buried three scrolls down in "Commands" in addition to appearing in Mission Control.
Critical
Duplicate name-entry flows create confusion
There are two separate hunter name input flows: the first-contact-backdrop modal (first run) AND the hunter-name-input-inline inside the HQ awakening gate section. A returning user who hasn't done daily login sees the awakening gate with a rename input — they may think they need to re-enter their name every day. The awakening gate section should not show name entry once first contact is complete.
Critical
"↺ NEW DAY SIM" button is visible in production UI
The end-day-sim-btn sits openly in the Commands row alongside Daily Login and End of Day. This is a developer testing tool. To any user, it signals the app is unfinished, in beta, or broken. It also breaks immersion instantly. This must be hidden — accessible only via a dev mode, a secret gesture, or the URL hash. This single button erodes trust more than any other element.
Critical
Daily login is a mandatory gate but the app doesn't explain why
A returning user who opens the app to quickly add a quest sees "SYSTEM OFFLINE" and can't interact until they hit Daily Login. There's no explanation visible — the Mission Control copy ("Login to unlock quests, earn XP, and protect your streak") only appears inside HQ after scrolling. The gate should be surfaced as a persistent banner or bottom sheet, not buried in a section the user has to find.
High
XP/progress data is represented 5 times on one screen
On HQ: stat grid cell (XP), stat grid cell (Level), stat grid cell (Streak), stat grid cell (Gold) + XP bar + Gold bar + Streak segments + Power bar. The same data is displayed in a 2×2 grid AND as progress bars. This wastes vertical space and dilutes the "aha" feeling of progress — spreading a dopamine hit across 5 elements instead of one powerful one.
High
EOD Summary panel is always rendered, showing zeros
The end-of-day-summary panel shows "Original XP: 0, Failed Quests: 0, Penalty: 0%" by default. A new user sees this and wonders if they already failed something. This should only appear after the End of Day action is triggered — hidden until needed.
High
Reset Profile sits next to Export/Import with no protective distance
Inside Data Management: Export Save | Import Save | ⚠ RESET PROFILE — three side-by-side buttons. The warning text below ("Reset applies permanent penalties") is vague. There's no separate confirmation modal, no input-to-confirm pattern. A user doing a routine export could accidentally tap Reset Profile on a small screen. Destructive actions must be isolated — different section, different visual weight, double confirmation.
Medium
Dungeon tab is locked with no visible explanation at nav level
The DNG tab shows as locked (tab-locked class) in the nav. The unlock requirement (Level 5 + Rank M) is only discoverable by somehow navigating inside the dungeon page. On mobile, a locked tab with no tooltip or badge showing progress is a dead end that creates confusion. A small "LVL 5 to unlock" sub-label under the tab button would fix this.
Medium
Bonus Gate is in the header AND the quest page with no explanation of what it is
A new user sees ◆ GATE pulsing in the header. They have no context. The gate mechanic — a bonus reward after completing all quests — is never explained in the UI. There's no tooltip, no first-run explanation, no contextual help. This is a compelling mechanic being wasted because users don't know it exists until they stumble into it.

Missing states

Empty quest list — no visible empty state. A new user on the Quests tab sees only the input field with no context about what quests do or examples.

No achievements until earned — the achievements grid renders empty with no "earn your first achievement" prompt.

No error handling visible for malformed JSON imports.

No "all quests complete" celebration state. After completing everything, the quest list just shows completed items. No victory moment.

The intro overlay plays every launch — there's no "don't show again" mechanism besides the skip button, which is re-shown every session.

Psychological analysis

Critical
Loss aversion is misfired — the Streak Shield neutralizes it
Streaks are the most powerful retention mechanic in habit apps (see Duolingo). They work through loss aversion: "don't break what you've built." But this app offers a Streak Shield item in the Arsenal shop that preserves streaks automatically on missed days. This converts loss aversion into false security. Users who buy the shield stop feeling urgency to log in. Duolingo sells streak freezes but gates them behind significant friction and shows them as emergency items — not products on a shelf.
High
No destination — users can't see where they're going
What does Level 2 give you? What changes at Rank M? What's in the dungeon once you unlock it? None of these are answered in the UI. The skill tree shows locked nodes (Strike at Level 2, Guard at Level 3) but doesn't communicate why these matter. Motivation requires a visible destination. The app creates a treadmill, not a journey.
High
The gold economy is confused — gold serves two incompatible purposes
The Gold progress bar in HQ is labeled "GOLD TOWARD RANK" — implying gold is used to rank up. But the Arsenal shop sells items for gold. If ranking up requires gold AND buying items requires gold, users face a resource competition they have no guidance on resolving. Is rank-up gold separate? Is item gold deducted from rank progress? This ambiguity kills spending decisions.
High
The UI assumes knowledge of Solo Leveling lore
Shadow Army, Gates, Dungeons, Rank M, Shadow Monarch, Hunter Designation — these are meaningful to Solo Leveling fans but opaque to anyone else. The app makes zero attempt to bridge the lore to the productivity use case. A new user who hasn't read the manhwa has no mental model. Even fans need occasional reminders of what in-app terms mean in productivity terms.
Medium
Passive UI — no urgency triggers during the day
The only urgency trigger is the EOD countdown clock in the header. But it's a passive number — it doesn't change color as the deadline approaches, doesn't pulse, doesn't send a notification. Duolingo's streak counter turns red and shows "at risk" when you haven't practiced. This app has the architecture for urgency (the countdown exists) but the emotional escalation is absent.
Medium
The intro overlay every launch erodes the "cool" factor fast
The awakening overlay ("INITIALISING PROTOCOLS...") is thrilling on day 1. By day 7 it is dead weight. The skip button helps but the user must actively dismiss it every session. A first-run-only intro with a quicker transition into the app would preserve the immersion without creating friction. Consider: short 0.5s boot animation vs. full overlay. Reserve the full sequence for actual level-up or rank-up moments.

Fixes by effort + impact

Quick wins — under 1 hour

15 minHigh impact
1. Remove the "NEW DAY SIM" button from the UI
Hide it behind a developer flag (?dev=1 URL param) or a long-press on the brand logo. Expected impact: immediate trust recovery for new users. This is the single fastest trust fix in the codebase.
10 minHigh impact
2. Change "SYSTEM OFFLINE" → "AWAKEN TO BEGIN"
Same semantic meaning, completely different emotional register. "Offline" signals broken. "Awaken to begin" is thematic and motivating. One string change. Impact: reduces new-user confusion and abandonment at first impression.
20 minHigh impact
3. Add icons and full labels to nav tabs
Replace ARS/HQ/QST/DNG with icon + word: ⚔ Shop · 🏠 Home · 📜 Quests · 🌀 Dungeon. Icons are processed faster than text. Expected impact: reduces navigation confusion for new users significantly, especially those not familiar with the game.
30 minHigh impact
4. Hide EOD summary by default — show only after End of Day fires
Add display:none to #end-of-day-summary by default. Toggle it visible only when End of Day action runs. Zero ambiguity for new users. Zero code change to logic — only state presentation.
20 minMedium impact
5. Add unlock tooltip to locked DNG tab
Show "Lvl 5 · Rank M" as a sub-label or tooltip on the locked dungeon tab. Users need to know what they're working toward at the navigation level, not just inside the dungeon page.
30 minMedium impact
6. Visually isolate the Reset Profile button
Move Reset to its own row below Export/Import, change its style to ghost-danger (outline only, red), and add a confirmation modal with "type RESET to confirm" before executing. Destructive action requires protective distance and friction.

Medium improvements — 1–2 days

1 dayHigh impact
7. Redesign HQ into 3 focused sections only
Keep: Status card (XP + streak + level in one compact row), Today's mission (mission control), and Quick Register. Everything else — Achievements, Skill Tree, Data Management — moves to a dedicated Profile/Settings tab. This removes 70% of HQ scroll and makes the primary action immediately visible. Expected impact: measurable drop in "daily login" completion time, higher daily activation rate.
4 hoursHigh impact
8. Move nav to bottom, use standard mobile tab bar pattern
Standard iOS/Android bottom nav with icon + label. Thumb-reachable. The current mid-screen placement is architecturally awkward on mobile. Expected impact: faster navigation speed, lower mis-tap rate, aligns with every major mobile app the user has ever used.
1 dayHigh impact
9. Add a "Next unlock" callout in the status card
Below the XP bar: "Strike ability unlocks at Level 2 — 88 XP away." Below the rank progress: "Dungeon unlocks at Rank M." Give users a visible destination at all times. This converts the XP grind from purposeless to purposeful. Directly inspired by how Duolingo shows what's in the next lesson tier.
6 hoursHigh impact
10. Replace onboarding checklist with an interactive 3-step card flow
The current plain-text list "1. Use Daily Login... 2. Add at least one quest..." is weak. Replace with 3 swipeable onboarding cards with visuals and a progress dot indicator. Auto-advance to next card on completion of each step. This transforms first-run from reading a manual into feeling like starting the game.
3 hoursMedium impact
11. Add urgency escalation to the EOD countdown
Under 4 hours remaining: countdown turns amber. Under 1 hour: turns red and pulses. Add a "X quests remain — complete them before midnight" text that appears when under 2 hours remain. This converts a passive clock into an active motivation trigger.

Major redesigns — high impact

3–5 daysHigh impact
12. Collapse status display from 5 bars to 1 "Hunter Card"
One compact card: Avatar/rank badge on the left, 3 stats inline (LVL / STRK / GOLD), one XP progress bar, and one "next unlock" line. That's all. Removes 4 redundant bars. The dopamine hit of filling a single bar is stronger than partially filling 5 bars simultaneously. This is the core loop visualization — it should feel satisfying, not bureaucratic.
1–2 daysHigh impact
13. Add a "Focus Mode" — quest-only full-screen view
When a user has active quests and is in completion mode, offer a stripped-down view: just the quest list, a completion counter, and the timer. No header, no nav. Just "clear all quests." Trigger it from a quest-page button: "Enter Focus Mode." After completing all quests, explode into the celebration sequence. This is how Duolingo structures lessons — the completion experience should be its own mode.
2–3 daysMedium impact
14. Clarify the gold economy with a glossary tooltip system
The "Gold Toward Rank" bar and the shop both use gold — clarify whether these are the same pool. Add tappable info icons (ⓘ) next to key mechanics (Gold, Rank, Shadow Army counter) that show a one-sentence explanation. This surfaces lore/mechanic context without cluttering the main UI. Makes the app accessible to non-Solo Leveling fans too.

Competitive benchmarks

Compared against Duolingo, Notion, and Spotify — the three top-tier apps in their categories most relevant to this app's design challenges.

vs. Duolingo (retention + gamification master)

Duolingo
Home screen shows exactly one primary action: "Continue" or "Start lesson." Zero ambiguity about what to do next. Every session starts with one tap.
This app
HQ shows 10+ sections. The primary action (Daily Login) requires scrolling to find in the Commands section, even though it also appears in Mission Control.
Duolingo
Streak UI is dead simple: one flame emoji with one number. It turns red with "at risk" language when the user hasn't practiced. The emotional escalation is calibrated precisely.
This app
Streak is shown in 3 places simultaneously. The EOD clock is passive. There's no emotional escalation as the deadline approaches. Streak Shield removes the consequence of breaking it.
Duolingo
Completion celebration is a full-screen XP animation with confetti. The reward is unmissable. The loop is tight: lesson → celebration → next lesson hint. Users know exactly what they accomplished and what comes next.
This app
Level-up event and rank-up event are well-designed visually. But day-to-day quest completion feedback is unclear — what happens when you check off a quest? Is there a per-quest celebration, or just XP accumulation?

vs. Notion (clarity of information architecture)

Notion
Empty states are inviting: "Start typing to create..." Every blank surface tells you what to do with it. Notion treats emptiness as potential, not absence.
This app
Empty quest list, empty achievements grid, blank image drop zones — multiple surfaces that look unfinished rather than ready. Empty states are missed retention opportunities.
Notion
Navigation is always visible and clearly labeled. Even first-time users can orient themselves within seconds because the information architecture is consistent and predictable.
This app
Navigation labels (ARS, HQ, QST, DNG) require prior knowledge of the app to parse. No icons. Mid-screen position fights mobile thumb ergonomics.

vs. Spotify (visual system discipline)

Spotify
Bottom nav: Home · Search · Library. Full words, clear icons, always reachable. Users never wonder where things are. The mental model forms in under 30 seconds.
This app
Mid-screen nav with 3-letter codes. DNG tab is locked without explanation at the nav level. Users have to explore to understand the structure.
Spotify
Two typefaces, maximum. The entire design language is built on two font weights and a strict color hierarchy. This is what makes Spotify look expensive.
This app
Four typefaces. Multiple button archetypes. Inline style overrides. The design system is inconsistent, which makes the app feel unpolished even when individual components are good.

Top 5 critical issues

1

NEW DAY SIM button visible to users

Single biggest trust killer. Makes the app look like a dev build. Fix in 15 minutes.

2

HQ tab information overload

10+ sections on one screen. No user can orient themselves. Primary action is buried. Restructuring this tab is the highest-leverage architecture change.

3

"SYSTEM OFFLINE" default state

Looks like a broken app. Immediate user abandonment trigger on first open.

4

Streak Shield undermines the retention mechanic

The streak is the primary retention driver. Selling an item that neutralizes streak loss at low cost converts urgency into complacency.

5

No visible destination — no journey, only treadmill

Users don't know what Level 2 gives them, what rank M means, or why they should care. Motivation requires a visible finish line, not just a progress bar.

Top 5 highest-ROI improvements

1

Collapse HQ to 3 sections + bottom nav

Highest leverage structural change. Makes the app feel polished, fast, and purposeful. Directly impacts daily activation rate.

2

Add "Next unlock" callout throughout

Converts abstract grind into purposeful progression. Shows users exactly what they're working toward. Turns every XP gain into a step toward something meaningful.

3

Add urgency escalation to EOD countdown

Converts a passive number into an active retention trigger. Color + language change when time is running out. Directly impacts quest completion rate and daily active usage.

4

Interactive onboarding card flow

Replaces text checklist with a game-like intro. Sets the RPG tone immediately. Dramatically improves day-1 retention — the most critical session in any habit app.

5

Nav label + icon fix

Smallest effort, immediately makes the app legible to anyone. Reduces new-user navigation confusion and first-session abandonment.
